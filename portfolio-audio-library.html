Audio Library (FastAPI + pCloud)

A backend system that allows uploading, storing, and streaming sermons seamlessly using FastAPI and pCloud API, with PostgreSQL for metadata.

FastAPI pCloud PostgreSQL REST API

Backend Development (FastAPI)

I was responsible for the backend architecture of the Audio Sermon Platform. My main role included building secure and scalable APIs, handling file uploads, and designing the database schema to store sermon metadata efficiently.

After researching file storage solutions, we used PostgreSQL for metadata storage and Cloudinary for audio and thumbnail hosting. I also implemented structured APIs for uploading, retrieving, searching, and streaming sermons.

It was my first time working with FastAPI so I did a project-based learning, just to get the basic Idea of what I wanna build and the basics of FastAPI.

This ensured that the frontend team could easily consume data while maintaining performance and scalability.

Backend Development Process

1

Research & Tools

I researched affordable cloud storage solutions and explored how to integrate them with the app. This included comparing services and finally working with pCloud for audio storage and PostgreSQL for metadata.

2

Learning FastAPI

This was my first time working with FastAPI. I learned how to create scalable APIs through project-based learning, starting with small practice projects that focused on request handling and routing.

3

Database Practice

I built a mini project using PostgreSQL to understand relational database design and queries. This helped me structure the sermon metadata model effectively before integrating it into the real app.

4

pCloud Integration

Since pCloud had closed their app creation system, I contacted their support team for solutions. Meanwhile, I experimented with smaller projects to connect FastAPI with pCloud‚Äôs API for file uploads and retrievals.

5

Final Integration

I combined PostgreSQL and pCloud into the main project, ensuring that sermons could be uploaded, stored, and retrieved smoothly. This was the foundation of the APIs delivered for Mr. Timothy‚Äôs project.
